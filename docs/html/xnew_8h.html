Detailed Description

Fixed-block pool equivalents of operator new / delete.

xnew<T>(args...) allocates from the xallocator fixed-block pool and constructs T in-place via placement new. xdelete<T>(p) calls the destructor then returns the memory to the pool via xfree().

These are the pool-aware counterparts of new(std::nothrow) / delete for types that do not carry the XALLOCATOR macro (e.g. user argument types marshalled through make_tuple_heap).
